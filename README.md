# 🧠 AI Self-Evolving Skills

> **让 AI 越用越聪明：自动经验沉淀系统**

## 🚀 为什么这个项目与众不同？

### ❌ 传统 Skill 仓库的问题

大多数 GitHub Skill 项目只是**静态的技能集合**：
- 📦 一堆预设好的指令文件
- 🔧 需要手动安装和配置
- 🔄 不会随着使用而进化
- 🤔 遇到新问题还是要重新思考

### ✅ 我们的突破：自我进化系统

这是一个**动态的、会学习的 AI 思维框架**：

```
遇到问题 → 自动监控 → 触发沉淀 → 生成 Skill → 下次复用
   ↑                                                        ↓
   └────────────────── 循环优化 ────────────────────────────┘
```

**核心特性**：
- ⚡ **自动触发**：无需人工干预，满足条件自动沉淀经验
- 🎯 **精准提炼**：剔除无效操作，只保留最优解决路径
- 📈 **持续进化**：每次解决问题都让系统更聪明
- 🔄 **优先复用**：已有 Skill 自动提升优先级，跳过重复思考

## 💡 核心机制

### 三大自动触发条件

| 触发类型 | 判定标准 | 典型场景 |
|---------|---------|---------|
| ⏱️ **时长触发** | 推理+调试耗时 > 40秒 | 复杂报错、链路异常 |
| 🔄 **试错触发** | 同一故障点试错 ≥ 3轮 | 参数调整、方案切换 |
| 🔗 **同源触发** | 识别为历史同类问题 | 端口冲突、消息流异常 |

### 五步沉淀流程

```
1️⃣ 复盘全流程 → 拆分有效/无效操作
2️⃣ 压缩精简 → 剔除多余步骤
3️⃣ 标准化成文 → 生成 Skill 文档
4️⃣ 优先级锁定 → 提升执行优先级
5️⃣ 禁止冗余 → 只固化最优解法
```

### 执行优先级规则

```
全局自检沉淀规则 > 临场自由思考
既定排错流程 > 发散分析猜测
故障处理动作 > 长时间逻辑推演
```

## 🛠️ 如何使用

### 前置要求

- Python 3.8+
- LamaPuppeteer 中间层服务
- Cursor IDE（或其他支持代理的客户端）

### 快速开始

1. **克隆仓库**
```bash
git clone https://github.com/drgon1/santian.git
cd santian
```

2. **启动服务**
```bash
cd lama-puppeteer
.\start_dispatcher.bat
```

3. **配置 Cursor**
在 Cursor 设置中添加自定义模型代理：
```
http://127.0.0.1:9100/v1
```

4. **开始使用**
正常使用 Cursor，系统会自动监控并沉淀经验。

### 查看已沉淀的 Skills

```bash
ls .lingma/skills/
```

## 🌟 设计理念

### 1. 不要重复造轮子

> "同样的问题不应该解决两次。"

每次遇到问题，系统都会检查是否有现成的 Skill。如果有，直接复用；如果没有，解决后自动沉淀。

### 2. 先行动，后思考

> "重启只需要 30 秒，分析可能需要 30 分钟。"

遇到故障时，优先执行既定动作（如重启），而不是陷入长时间的逻辑推演。

### 3. 经验即资产

> "每一次调试都是对系统的投资。"

所有的调试经验都会被转化为可复用的 Skill，形成团队的知识资产。

## 🤝 贡献指南

欢迎贡献新的 Skills 或改进现有机制！

### 添加新 Skill

1. Fork 本仓库
2. 创建新的 Skill 文件：`.lingma/skills/你的skill名称.md`
3. 按照标准格式编写：
   ```markdown
   # [问题名称] 解决方案
   
   ## 适用场景
   [什么情况下会触发这个问题]
   
   ## 故障现象
   [具体的错误表现]
   
   ## 一键处理方案
   [最简固定解决步骤，可直接复制执行]
   
   ## 原理说明（可选）
   [简要解释为什么这样能解决]
   ```
4. 提交 Pull Request

如果发现有 Bug 或有改进建议，请提交 Issue。
