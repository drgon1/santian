    需求拆解项目搭建规划技术选型分析
  • gstack-multi-agent

    需要DOM解析/语义化元素定位多策略选择器生成浏览器自动化容错路由
  • karpathy-skills
    Andrej Karpathy 编程行为准则（精简：精准修改、不废话、结果验证）。触发条件：代码精准修改、最小改动需求、精准重构、结果验证。不匹配常规BUG排错场景（由mattpocock-skills处理）。
    代码精准修改最小改动需求精准重构

  • mattpocock-skills
    Matt Pocock 编程Skill合集（核心：排错、极简输出、架构规范）。触发条件：BUG调试排错、代码诊断、性能回归排查、极简输出需求、架构重构优化、代码库解耦、模块深度设计、TDD测试驱动开发。
    BUG调试排错代码诊断性能回归排查

    多用户并发风控处理账号池管理
  • slider-captcha-solver

    滑块验证码NC captcha滑动拼图

  • wechat-cookie-bridge

🌐 浏览器自动化类
  • browser-agent-debug
    Browser Agent 调试方案（通用版）。触发条件：需要调试浏览器自动化脚本、验证发布流程、修复元素选择问题。核心原则：CDP连接→DOM查询→直接操作→DOM验证，免登录调试。支持所有 AI 编程工具（Lingma/Cursor/Claude Code等），通过 CDP 连接到 CloakBrowser。
    需要调试浏览器自动化脚本验证发布流程修复元素选择问题。核心原则：CDP连接→DOM查询→直接操作→DOM验证
  • browser-autonomous-agent
    浏览器自主智能体 - 基于 Playwright CDP + CloakBrowser 的 AI 自主浏览器自动化方案。输入自然语言任务，AI 通过 CDP 协议直接查询 DOM、自主决策操作、执行任务并验证结果。整合反检测能力，可登录各大自媒体平台（头条/百家/大鱼/微信/小红书等）。触发条件：任何需要浏览器自动化完成的任务、数据采集、发布操作、页面操作。
    任何需要浏览器自动化完成的任务数据采集发布操作
  • browser-interaction-flow
    浏览器操作流程正确设计方法论。包含全量DOM分析、操作→检测循环、效果验证。在涉及浏览器页面交互、按钮点击、流程设计时必须先调用本Skill加载规范。
  • browser-provider-implementer
    网页端大模型对接路由开发全流程。触发条件：浏览器网页模型对接、CloakBrowser转发、内网穿透端口配置、IDE代理调试、接口通信异常修复。兼容豆包/千问/Kimi/DeepSeek等所有网页端模型。
    浏览器网页模型对接CloakBrowser转发内网穿透端口配置
  • cloak-not-playwright
    禁止直接调用 Playwright SDK，强制使用 CloakBrowser 包装层。触发条件：编写/修改涉及浏览器自动化代码、导入 playwright、调用 page._page、使用 raw_page 时。
    编写/修改涉及浏览器自动化代码导入 playwright调用 page._page

如何使用

方式 1：直接在 GitHub 查看

点击上方的 Skill 链接，直接在 GitHub 上阅读完整文档。

方式 2：克隆到本地使用

git clone https://github.com/drgon1/santian.git
cd santian/ai-skills/.lingma/skills

方式 3：导入到 Lingma IDE

将整个 Skill 文件夹复制到你的 .lingma/skills/ 目录即可使用。
